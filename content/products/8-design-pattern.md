This is a **Design Patterns study guide** covering 8 classic Gang of Four patterns in C#:

1. **Decorator** – dynamically adding responsibilities to objects
2. **Composite** – tree structures for part-whole hierarchies
3. **Adapter** – converting incompatible interfaces
4. **Prototype** – cloning objects from a template
5. **Singleton** – ensuring a single global instance
6. **Strategy** – swappable algorithms at runtime
7. **Chain of Responsibility** – passing requests along a handler chain
8. **Command** – encapsulating requests as objects

Each pattern includes an **intent explanation**, **UML class diagram**, **sample C# code**, and **3 real-world examples** (e.g. coffee shop, notification system, smart home, restaurant ordering).
